void atomic_bit_clear (  volatile void *  addr,
int  nr
)  [inline, static]
 

Clears a bit in memory.

Parameters:
nr Bit to clear.
addr Address to start counting from.
atomic_bit_clear() is atomic and may not be reordered. However, it does not contain a memory barrier, so if it is used for locking purposes, you should call smp_mb__before_clear_bit() and/or smp_mb__after_clear_bit() in order to ensure changes are visible on other processors.

void atomic_bit_set (  volatile void *  addr,
int  nr
)  [inline, static]
 

Atomically set a bit in memory.

Parameters:
nr The bit to set.
addr The address to start counting from.
This function is atomic and may not be reordered. See __set_bit() if you do not require the atomic guarantees. Note that nr may be almost arbitrarily large; this function is not restricted to acting on a single-word quantity.
